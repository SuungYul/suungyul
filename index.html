About

프론트에서 시작해 풀스택·데이터·인프라까지

React·Flutter 개발자로 시작했고, 지금은 제약·헬스케어 B2B(CRM·세일즈/마케팅) 제품을 DB 설계부터 앱·배치·운영까지 직접 맡고 있습니다.

"넘겨받은 코드를 다시 설계해 끌어올렸습니다." 앱 개발 체험형 인턴으로 들어와 정규직으로 전환되면서, 제품 전반을 맡게 됐습니다.

처음 넘겨받은 코드는 구조가 거의 없었습니다. 파일 4개에 몰려 있던 FastAPI 서버를 레이어드 아키텍처로 다시 나눴고, 자주 끊기던 데이터 배치는 Airflow로 새로 짜서 안정화했습니다. 그 위에서 백엔드·모바일·관리자 웹·데이터·운영 인프라를 직접 맡고, 느린 쿼리는 원인을 찾아 수십에서 수백 배까지 끌어올렸습니다.

AI는 적극적으로 쓰지만, 무엇을 어떤 구조로 만들지는 직접 정합니다. 아키텍처와 우선순위, 트레이드오프는 제가 판단하고 Claude·Codex로 구현 속도를 높입니다. 작업마다 Codex로 코드 리뷰를 받고, 기획·마케팅·디자이너와 함께 일하며, 비개발자도 데이터를 직접 볼 수 있게 분석 가이드를 만들어 둡니다.

🧱

레이어드 아키텍처

꾹·탁 백엔드 모두 routers → services → repositories → models 4계층으로 나눴습니다. 라우터는 검증·매핑, 서비스는 도메인 로직, 리포지토리는 SQL을 맡습니다. 역할이 나뉘어 있어 테스트와 수정이 쉽습니다.

🔌

교체 가능한 DB 추상화

mariadb·snowflake 모듈이 동일한 공개 API(init/get/close pool)를 제공하도록 설계. 덕분에 Snowflake → MariaDB 전면 이관을 라우터·서비스 수정 없이 import 교체만으로 완료.

⚡

성능을 코드로 증명

옵티마이저가 60억 row BNL 조인으로 폭발하던 쿼리를 다단계 CTE로 분리해 PK·인덱스를 타게 재작성. 잘못된 async로 6분 걸리던 API의 병목 제거. 느리면 먼저 EXPLAIN으로 원인부터 확인합니다.

🛡️

방어적인 백엔드

JWT 파싱 + 민감정보(password·uuid·ip) 자동 마스킹 로깅 미들웨어, SQL 식별자 화이트리스트 + placeholder 바인딩으로 주입 차단, ALB health-check 로그 분리, 대용량 응답 GZip ~85% 압축.

🔁

프론트 비동기 경쟁 제어

AbortController + 시퀀스 토큰으로 stale 응답을 드롭하고, IntersectionObserver 무한 스크롤 + 디바운스, React Strict Mode 이중 마운트 버그까지 코드로 대응(useCandidates).

📱

모바일 상태 일관성

Flutter GetX MVC(view·controller·service·store 분리) + 찜 상태를 낙관적 업데이트 3단계(local→confirmed→rollback)로 동기화. 공용 위젯 추출과 Freezed 불변 모델 + 하위호환 역직렬화.

-- 거의 모든 조회가 BZMN_NO 로 JOIN 되는데 인덱스 부재 → 풀스캔
EXPLAIN SELECT a.*, b.SLS_NOCS
  FROM DS_NTGTN_RPRT_MST a
  JOIN AFFL_CNPT_TRS b ON a.BZMN_NO = b.BZMN_NO;   -- type=ALL (Full Scan)

-- 조인 키에 인덱스 추가 (EXPLAIN으로 풀스캔 확인 후)

마케팅 담당자에게 "어떤 병·의원을 공략해야 하는지" 추천하는 타겟 마케팅 서비스입니다. 심평원·매출·진료과 데이터로 타겟을 뽑아 캠페인으로 연결합니다.

핵심 기여
  • 타겟 추출 엔진과 UX: 변수 카탈로그, 값 검색 미리보기(디바운스 + 무한 스크롤), 드래그 없이 쓰는 필수/선호/제외 버튼, 등급별(RBAC) 데이터 가시성.
  • Snowflake → MariaDB 전면 이관 — DB 추상화 계층 덕에 라우터·서비스 수정 없이, 29테이블 + 11뷰 + 심평원 9테이블 47개 파일 repoint, SQL 방언 변환, 콜레이션 함정 수정.
  • 필터 조건을 SQL로 번역하는 filter_sql 엔진 — must/any/not 버킷을 AND/OR/AND NOT로, 식별자 화이트리스트로 주입 방어. 핵심 뷰 실체화(CTAS+인덱스)로 타겟 COUNT 60초+ → 0.2초 (8만+ 병원 뷰)
  • 사용자 데이터 업로드 / 병원 매핑 전체 구현: 16개 엔드포인트, 5단계 마법사, 이름·주소 유사도 매칭.
  • EC2 x86 → ARM 무중단 이관(ENI/IP swap, CI 러너 재등록) 연 ~71% 비용 절감, 자동 로그아웃 + 비밀번호 재발급(SMTP 디버깅).
